Advanced Data Structures

The objective of the assignment is to: (1) explore advanced data structures, and (2) benchmark algorithms.

Consider the list of advanced data structures provided at https://iq.opengenus.org/list-of-advanced-data-structures/ and at https://en.wikipedia.org/wiki/List_of_data_structures. Select 3 comparable data structures.

Submission

To complete the assignment, submit the following:
  1. A concise pdf report including
    • the time and space computational complexity of your data structures,
    • a table summarizing the measured performance,
    • a performance profile plot (see below),
    • comments on whether the switching phenomenom occurs in your performance profile (see performance profile limitation below),
    • an explantions of why you selected those data structures (what attracted them; this is totally subjective and personal).
    Use appropriate parameters to describe the complexity, e.g. Bloom filter uses n (estimated number of elements/logins), m (bit array of m bits), and k (number of hash functions). You do not need to justify the complexity; only to explain each parameters used and provide the formula with appropriate references.
  2. MATLAB code comparing the data structures selected. Because there are more than 2 data structures, read about performance profiles definition, limitations, and application to machine learning. MATLAB code for creating profiles is available in the provided perf.m file. You are responsible for generating appropriate data sets for the performance profile. Provide any justification of your choices in the report. Do not hesitate to contact the instructor (a quick email can save you hours for misguided work!).

    As for Assignment 1, your code should

    • take advantage of MATLAB Object-Oriented language features
    • use MATLAB unit testing framework based on Write Simple Test Case Using Classes
    • use MATLAB CodeCoveragePlugin class to generate a coverage report, see CodeCoverage pluging. Alternatively you can generate the coverage report using MATLAB profiler, see Determine Code Coverage Using the Profiler (however, this is a manual process so everytime you need to regenerate the report, it requires several mouse clicks).
    • follow common practises:
      • appropriate class/variable/method names (not too long and meaningful),
      • appropriate comments using MATLAB, see Add Help for Your Program
      • comments for each method should indicate input, output, and a short explanation of the method
      • clear setup and running instructions.
      An example of acceptable practises is the MATLAB code for the CCA2 numerical library.

Submit your .zip file to Canvas.
